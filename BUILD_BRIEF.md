## ПРОДУКТ (одним абзацем)

AlemPrep — бесплатная веб-платформа подготовки к ЕНТ (Казахстан), предметы: математика,
информатика, физика. Стек: Next.js 15 (App Router) + React 19 + TypeScript strict + Tailwind
+ Supabase (Postgres/Auth/RLS/SSR) + next-intl (ru/kk). UX как oneprep.xyz: минимализм,
тёмная тема, клавиатурные шорткаты, KaTeX для формул. Полный контекст — в CLAUDE.md и MVP_PLAN.md.

## ЗАФИКСИРОВАННЫЕ РЕШЕНИЯ (не переобсуждать)

- AI для контента — **Claude Haiku** через `@anthropic-ai/sdk`. Gemini выпиливаем.
- Бэкенд — **Supabase**, остаётся.
- Заливка задач — **прямо в БД** через `SUPABASE_SERVICE_ROLE_KEY`, `is_published=false`.
  Никаких `.sql`-файлов для ручной вставки.
- **ИИ-верификатора НЕТ.** Качество проверяет человек в review-странице. Оставить только
  детерминированные проверки: zod + дедуп (одинаковые числа) + санити LaTeX.
- Порядок по картинкам: math+информатика (текст) → T11 (картинки) → физика (с картинками).
- Скоуп MVP: математика ~200–250, информатика ~120–150, физика ~120–150.

## РАБОЧИЕ ПРАВИЛА

- Один таск = один коммит с понятным сообщением.
- После каждого таска — `/verify` (typecheck + lint + build). Красный чинишь до зелёного.
- НЕ трогать: дизайн-токены (`app/globals.css`, `tailwind.config.ts`), `.env.local`,
  уже применённые миграции. Новая схема — только новыми файлами миграций.
- Тексты — синхронно в `messages/ru.json` И `messages/kk.json` (паритет ключей). Казахский —
  нормальный перевод, не калька; сомнительное помечай `TODO`.
- TypeScript strict, без `any`/`@ts-ignore` без причины. Server Components по умолчанию.
- Запись в БД — через server actions/service-role; чтение — через `lib/supabase/queries`.

## ПОСЛЕДОВАТЕЛЬНОСТЬ (эпики)

### Эпик A — Автопайплайн на Haiku ⭐
Переписать контент-пайплайн с Gemini на Anthropic Haiku.
1. `scripts/transcribe-questions.ts`: `@google/genai` → `@anthropic-ai/sdk` (Haiku vision).
   PNG → reference JSON. Сохранить zod-валидацию и пометку `skip` для графиков.
2. `scripts/generate-variants.ts`: на Haiku, structured output + zod. Если у эталона есть
   картинка — 1–2 варианта, иначе N.
3. Детерминированные проверки (НЕ ИИ): zod-схема + дедуп по числам/stem + санити LaTeX.
4. Новый `scripts/insert-to-db.ts`: запись прошедших прямо в Supabase через
   `SUPABASE_SERVICE_ROLE_KEY` (серверный клиент, минует RLS), `is_published=false`.
5. npm-скрипт `gen:all -- --dir <папка> --subject <slug>`: транскрипция → генерация →
   проверки → заливка одной командой.
6. Лог input/output токенов + примерная стоимость партии в USD (расценки Haiku).
7. Удалить Gemini-код и зависимость `@google/genai`; старые `scripts/generated|references`
   → `scripts/_archive/`. Задачи в БД не трогать.
Демо-прогон на 3 эталонах, показать, что заливка прошла. Коммит. `/verify`.
