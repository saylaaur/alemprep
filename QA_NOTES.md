## 3. Vitest + юнит-тесты критической логики

**Сделано (инфраструктура):**
- Vitest 4 (`npm i -D vitest --legacy-peer-deps`), `vitest.config.ts` с алиасом `@`, скрипт `npm test`.
- Извлечена чистая логика из компонентов/actions, чтобы её можно было тестировать:
  - `lib/practice.ts` — `checkAnswer` / `isAnswerComplete` (были приватными в `PracticeView.tsx`);
  - `lib/streak.ts` — `advanceStreak` / `localDateStr` / `previousDateStr` (стрик был инлайном в `recordAttempt`).
- Тесты: `lib/exam.test.ts` (scoreAnswer — 24 кейса), `lib/streak.test.ts`, `lib/practice.test.ts`. Итого 44 теста.

**Найденные и починенные баги:**
1. **`scoreAnswer` (matching): пустой ответ давал частичный балл.** `{}` или `{a: ''}` при вопросе с одной парой считались «одной ошибкой» → 1 балл вместо 0. Теперь ответ без единой заполненной пары — это пропуск (0), как и пустой выбор в multi. Поймано тестом до фикса.
2. **Стрик считался по UTC-дате, дневная цель — по локальной полуночи.** `recordAttempt` брал `toISOString().slice(0,10)` (UTC), а `getTodayAttemptsCount` — локальную полночь. На сервере в TZ ≠ UTC активность около полуночи попадала в разные «дни». Теперь оба на локальной дате (`localDateStr`), вычисление «вчера» — TZ-независимое.
3. Попутно: `checkAnswer('multi', не-массив)` раньше падал бы на `.length` у строки — извлечённая версия строго проверяет типы (было маскировано кастом `as string[]`).

## 4. Логические баги и edge-cases по фичам

**Пробник (`MockExamView`):**
1. **Молчаливый провал старта.** Если `createExamSession` возвращал ошибку (нет сети, не залогинен, пустой `subjectId`), `startExam` делал `return` — кнопка «Начать пробник» просто не срабатывала без какого-либо фидбэка. Теперь пробник стартует и без сессии: баллы считаются на клиенте, не сохранятся только попытки в БД (см. «Открытые вопросы»).
2. **Пустой ответ считался «отвечено».** `setAnswer` всегда ставил флаг `answered`, даже когда ответ становился пустым (снял все галочки multi → `[]`; вернул селект matching на плейсхолдер → `{a:''}`). Навигатор и диалог подтверждения показывали такие вопросы как отвеченные, а в результатах они шли в «неверно», а не «пропущено». Введён `isAnswerEmpty()` (lib/practice + тесты): флаг ставится по фактической пустоте, `skipped` в результатах — тоже.
3. **Смена ответа сбрасывала пометку «вернуться позже».** Теперь `flagged` при изменении ответа сохраняется.

**Прогресс:**
4. **Heatmap сдвигался на день в TZ восточнее UTC.** Ячейки сетки и ключи активности строились через `toISOString()` (UTC-дата), при этом «сегодня» — локальное. В Казахстане (UTC+5/+6) вся активность рисовалась днём раньше. Теперь обе стороны на `localDateStr()` — общий базис со стриком и дневной целью.

**MathText:**
5. **Утечка ретрай-цикла.** Если KaTeX ещё не загрузился, `tryRender` перезапускал себя через `setTimeout` каждые 100 мс без cleanup — циклы жили после смены вопроса и анмаунта. Добавлен `clearTimeout` в cleanup эффекта.

**Проверено, багов не найдено:** страницы subjects/topic (notFound на неизвестный slug), admin review (redirect для не-админа, ошибки actions видны как error page), login, AppShell, dashboard (деление на 0 защищено `?? 20`), progress empty-state.

## 5. Доступность и клавиатура

**Найдено:**
- `globals.css` глобально ставит `:focus-visible { outline: none }`, а ring возвращают только `ui/button` и один select — **все кастомные интерактивные элементы были без видимого фокуса** (варианты ответов, навигаторы вопросов, кнопка фокус-режима, флаг, drawer, тумблер темы, ссылки сайдбара, аккордеон результата). globals.css не трогали (запрещено) — ring добавлен по компонентам классами `focus-visible:ring-4 focus-visible:ring-ring/25` (конвенция из ui/button).
- Хинт шорткатов в тренажёре был захардкожен «1–4», хотя вариантов бывает 5–6 (multi), а обработчик принимает 1–9. Теперь показывает реальное число вариантов и скрыт для matching.
- Drawer не закрывался по Escape; не был размечен как диалог.
- Селекты matching не имели подписи для скринридера.

**Сделано:**
- Focus-ring на всех интерактивных элементах: PracticeView (варианты single/multi, навигатор, фокус-режим), MockExamView (варианты, оба навигатора, флаг, аккордеон), MobileNav (бургер, крестик, ссылки, выход), Sidebar (ссылки, выход), theme-toggle.
- aria: `aria-pressed` (выбор вариантов, флаг, фокус-режим), `aria-current` + `aria-label` на кнопках навигаторов, `aria-expanded` на аккордеоне результата, `aria-label` на селектах matching (ключи `matchingSelectFor` ru+kk), `role="dialog" aria-modal` на подтверждении завершения пробника и drawer.
- MobileNav: Escape закрывает drawer.
- Клавиатура по типам вопросов: 1–9 выбирают вариант в single/multi (обе вьюхи), стрелки ←/→ листают вопросы везде, Enter в тренажёре проверяет/идёт дальше; для matching цифры намеренно не активны (выбор через Tab + стрелки в селектах), стрелки в фокусе селекта не перехватываются.

## Открытые вопросы (нужно решение владельца)

1. **Пробник без сессии в БД.** После фикса молчаливого старта пробник запускается даже если `createExamSession` упал (нет сети/не залогинен) — результат считается на клиенте, но попытки не сохраняются, и пользователь об этом не узнаёт. Показывать предупреждение? Нужен UX-текст (ru+kk).
2. **Пробник не двигает стрик.** `recordAttempt` (тренажёр) обновляет стрик, а `finishExamSession` (пробник) — нет: 40 решённых на пробнике задач не продлевают серию. Баг или замысел?
3. **Обновление страницы во время пробника теряет весь прогресс** (state только на клиенте, таймер сбрасывается). Персистить в sessionStorage/БД — отдельная фича.
4. **Drawer без focus-trap.** Escape и клик по оверлею работают, aria-разметка есть, но Tab может уйти под оверлей. Полный trap — либо ручной код, либо зависимость (radix-dialog). Стоит ли?
5. **kk-контент пробника.** `getMockExamQuestions` фильтрует по `language = locale`: у казахской локали пробник будет пустым, пока нет kk-задач в банке (сейчас контент ru).
6. **CLAUDE.md устарел** в двух местах: «строки в PracticeView захардкожены» (уже нет) и «`npm run build` — только она проверяет типы» (теперь есть `npm run typecheck` с обоими конфигами + `npm test`). Обновить при случае.
7. **`app/auth/callback` хардкодит `/ru`** — известный тех-долг (ROADMAP §1.3), не трогал: завязан на деплой.
