# AlemPrep — Спринт MVP (рабочий бриф для Claude Code)

> Тактический бриф под недельный спринт. Стратегия — в `MVP_PLAN.md`, код-гайд — в `CLAUDE.md`.
> **Как пользоваться:** копируй блок «Промпт» в Claude Code. Один таск = один коммит.
> Перед «готово» гоняй `/verify` (typecheck + lint + build). Зелёное — иначе не закрываем.

---

## Зафиксированные решения (не переобсуждаем)

- AI контента — **Claude Haiku 4.5** (`@anthropic-ai/sdk`), не Gemini.
- Бэкенд — **Supabase**, остаётся.
- Заливка — **прямо в БД** через `SUPABASE_SERVICE_ROLE_KEY`, `is_published=false`. Без ручного SQL.
- **ИИ-верификатор не делаем.** Проверяет человек через review-страницу. Оставляем только
  zod + дедуп + санити LaTeX.
- Скоуп: математика + информатика + физика. Порядок по картинкам — см. `MVP_PLAN.md §3`.

---

## Стратегия субагентов (как гнать за неделю)

Эпики делятся на **независимые треки по файлам** — их безопасно гнать параллельными агентами:

| Трек | Эпики | Трогает | Конфликтует с |
|---|---|---|---|
| **Контент** | A, B, C, D | `scripts/`, `supabase/migrations/`, review-страница | — |
| **Ядро UI** | E (T5, T7) | `components/`, `app/.../full-practice`, `messages/` | — |
| **Деплой** | G | Vercel/конфиги | делается последним, после E |

Правила параллельной работы:
- Запускай два агента в **разных git-worktree** (Claude Code это умеет), чтобы не было
  конфликтов коммитов. Трек «Контент» и трек «Ядро UI» не пересекаются по файлам.
- Внутри трека — **строго последовательно** (A → B → C → D), есть зависимости.
- Каждый агент: один таск = один коммит с понятным сообщением + `/verify` перед закрытием.
- **Не трогать** дизайн-токены (`globals.css`, `tailwind.config.ts`), `.env.local`,
  применённые миграции. Тексты — синхронно в `ru.json` И `kk.json`.

Рекомендуемый запуск: сначала миграции тем (B1, B2 — быстрые, дают topic_slug пайплайну),
дальше параллельно агент-Контент (A→B→C→D) и агент-UI (E1→E2). G — в конце вручную.

## ТРЕК КОНТЕНТ

### A. Автопайплайн на Haiku ⭐ первый

**Готово, когда:** одна команда из папки PNG наполняет БД задачами (`is_published=false`)
без единого ручного SQL; лог стоимости в долларах выводится.

**Промпт:**
```
Перепиши контент-пайплайн с Gemini на Claude Haiku. Сначала план (plan mode), потом код.
1) scripts/transcribe-questions.ts: замени @google/genai на @anthropic-ai/sdk (Haiku 4.5,
   vision). PNG → reference JSON. Сохрани zod-валидацию и пометку skip для графиков.
2) scripts/generate-variants.ts: на Haiku, structured output + zod. Если у эталона есть
   картинка — генерируй 1–2 варианта (не N), иначе N.
3) Детерминированные проверки (НЕ ИИ): zod-схема + дедуп (отбраковка вариантов с теми же
   числами/stem) + санити LaTeX.
4) Новый scripts/insert-to-db.ts: пиши прошедшие в Supabase напрямую через
   SUPABASE_SERVICE_ROLE_KEY (серверный клиент, минует RLS), is_published=false. Никаких .sql.
5) npm-скрипт gen:all -- --dir <папка> --subject <slug>: транскрипция → генерация →
   проверки → заливка одной командой.
6) Логируй input/output токены и примерную стоимость партии в USD по расценкам Haiku.
7) Удали Gemini-код и зависимость @google/genai; перенеси старые scripts/generated|references
   в scripts/_archive/. Задачи в БД не трогай.
Ключи ANTHROPIC_API_KEY и SUPABASE_SERVICE_ROLE_KEY читай из .env.local (предупреди, что
Anthropic API — отдельный платный счёт). Демо-прогон на 3 эталонах. В конце /verify.
```

### B. Темы + наполнение (текст)

**B1 — темы информатики.** Промпт:
```
Создай миграцию supabase/migrations/0004_seed_informatics_topics.sql: 5–7 тем информатики
по спецификации НЦТ (системы счисления, алгоритмы и блок-схемы, программирование, базы
данных, сети, логика/булева алгебра). Привяжи к subject informatics. Обнови types/db.ts при
необходимости. Не правь старые миграции. /verify.
```

**B2 — темы физики.** Промпт:
```
Создай миграцию supabase/migrations/0005_seed_physics_topics.sql: ~8 тем физики по
спецификации НЦТ (механика: кинематика/динамика, молекулярная физика, термодинамика,
электростатика, постоянный ток, магнетизм, оптика, атомная/ядерная). Привяжи к subject
physics. /verify.
```

**B3/B4 — прогоны** (после A): `npm run gen:all` на папках math и informatics. Это запуск
команды, не кодинг. Заказчик проверяет результат в review-странице (D).

### C. Картинки + физика

**C1 — T11 поддержка картинок.** Промпт:
```
Добавь поддержку изображений в задачах. Сначала план (plan mode).
1) Схема: опциональное поле image?: {url: string; alt: string; caption?} в QuestionBody и в
   строке contexts (для контекстных заданий). Миграция + обновить types/db.ts и zod-схему в scripts/lib/schema.ts.
2) Хранение: Supabase Storage (публичный бакет questions-images), загрузка из пайплайна
   транскрипции — выгружать PNG задачи/контекста и проставлять url.
3) Рендер: показывать картинку над опциями в components/practice (и в контексте). next/image.
Дизайн-токены не трогай. Локализуй подписи ru/kk. /verify.
```

**C2 — прогон физики** (после C1): `gen:all --subject physics` уже с картинками.

### D. Review-страница

**Промпт:**
```
Сделай страницу /admin/review (защити: только profiles.is_admin=true; добавь колонку через
миграцию, если её нет). Показывай неопубликованные задачи (is_published=false): stem, опции,
правильный ответ, разбор, картинку. Дай кнопки «Опубликовать» (по одной и пачкой) и «Удалить».
Публикация = update is_published=true через server action. Локализуй ru/kk. /verify.
```

## Definition of Done (MVP)

- [ ] Пайплайн на Haiku наполняет БД одной командой, без ручного SQL
- [ ] ≥200 задач математики, ≥120 информатики, ≥120 физики (с картинками)
- [ ] Review-страница: публикация без SQL
- [ ] Мобильная версия работает (smoke-тест iOS/Android)
- [ ] Полный пробник проходится end-to-end
- [ ] Дизайн утверждён (полировка или редизайн)
- [ ] Задеплоено на публичный URL, OAuth работает в проде
- [ ] `/verify` зелёный, ключи ru/kk в паритете
