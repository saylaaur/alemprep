# AlemPrep — Стратегический план

*Версия 1.0 · Май 2026*

Полный план развития платформы подготовки к ЕНТ для рынка Казахстана. Документ обновляется в конце каждого спринта.

---

## 0. Executive Summary

**AlemPrep** — бесплатная веб-платформа для подготовки к ЕНТ (Единое национальное тестирование, Казахстан). Фокус: математика, физика, информатика. UX в духе oneprep.xyz: минимализм, отсутствие отвлечений, клавиатурные шорткаты, дарк-мод.

**Текущая позиция (май 2026):** работающий MVP на localhost. Полная инфраструктура: Next.js + Supabase + Google OAuth + i18n (ru/kk). 1 предмет активен (математика), 12 тем по спецификации НЦТ, 9 задач по теме «Логарифмы» (все 4 типа из реального ЕНТ).

**Цель на 12 месяцев:** 10 000 еженедельно активных учеников в Казахстане к маю 2027 (пик сезона ЕНТ). Полное покрытие математики + физики + информатики. ~1500 проверенных задач. Деплой на публичный URL.

**Цель на 36 месяцев:** платформа №1 для подготовки к ЕНТ в Казахстане. Покрытие всех 13 профильных комбинаций. Маркетплейс репетиторов. Возможная экспансия: Узбекистан (ДТМ), Кыргызстан (ОРТ).

**Шаг 1.5 — Аутентификация (✅ закрыт):**
- Supabase SSR-клиенты (browser + server)
- Server actions: `signInWithGoogle`, `signOut`
- Google OAuth провайдер настроен в Supabase + Google Cloud Console
- OAuth callback на `/auth/callback`
- Middleware-guard: редирект на `/login` для защищённых маршрутов
- Сессия в HttpOnly cookie через `@supabase/ssr`

**Шаг 2 — База данных (✅ закрыт):**
- 7 таблиц: `profiles`, `subjects`, `topics`, `contexts`, `questions`, `sessions`, `attempts`
- 2 ENUM: `question_type` (single/multi/matching), `session_mode` (practice/topic_drill/mock_exam)
- RLS-политики на каждой таблице: контент читают `authenticated`, personal — только свой
- Триггер `on_auth_user_created` для автосоздания profile + backfill для существующих юзеров
- Триггер `profiles_updated_at` для авто-update timestamp
- Расширение `pgcrypto` для `gen_random_uuid`
- 3 миграции: schema, seed-subjects, seed-logarithms
- Seed: 3 предмета, 12 математических тем, 1 контекст (pH), 9 задач (5 single + 1 multi + 1 matching + 2 context-sub)

**Шаг 3 — Тренажёр (✅ MVP):**
- Полиморфный `QuestionRenderer` для трёх типов (single/multi/matching)
- KaTeX через CDN — рендер $\ln^2 x - 3\ln x = 0$ и более сложных формул
- `PracticeView` client component с локальным state (Zustand не понадобился)
- Контекстный блок: общий текст показывается над сабвопросами
- "Проверить → объяснение" механика — explanation скрыт в DOM до раскрытия
- Сетка номеров 1–N с цветовой подсветкой (правильно/неправильно/текущий)
- Прогресс-бар сверху
- `recordAttempt` server action — пишет в `attempts` + обновляет стрик в `profiles`
- Стрик-логика: +1 если решал вчера, =1 если пропуск ≥2 дней
- `revalidatePath` после попытки — дашборд показывает свежие цифры
- Клавиатурные шорткаты: `1-6` выбор, `Enter` проверить/дальше, `←/→` навигация
- Подсказка по клавишам в UI
- ThemeToggle в Sidebar — переключение light/dark
- Sign Out форма

## 2. Целевая аудитория и рынок

### 2.1 Размер рынка

**Численно:** ~190 000 школьников сдают ЕНТ каждый год в Казахстане (по данным НЦТ за 2023–2025). Из них:
- ~80% сдают профильную математику (~150 000)
- ~30% сдают физику (~57 000)
- ~25% сдают информатику (~47 000)

**TAM (Total Addressable Market):** 190 000 выпускников/год + 200 000 десяти/одиннадцатиклассников готовящихся заранее = ~400 000 активных учеников в любой момент времени.

**SAM (Serviceable):** учитывая что нам нужен интернет и устройство — ~85% покрытия, итого ~340 000.

**SOM (Realistic year 1):** 10 000 учеников за первый год = 3% от SAM. Реалистично при правильном go-to-market.

### 2.2 Сегментация

| Сегмент | Доля | Особенности |
|---|---|---|
| Город, рус. яз | 30% | Высокая платежеспособность, часто уже на платных платформах |
| Город, каз. яз | 20% | Растущий сегмент, мало качественных ресурсов |
| Село, рус. яз | 25% | Низкая платежеспособность, плохой интернет |
| Село, каз. яз | 25% | Самый недообслуженный сегмент — наш приоритет |

**Наш сильный сегмент:** село + казахский. Конкуренция там минимальна, цена free решает.

### 2.3 Персоны

**Айгерим, 17, Актобе, 11 класс**
- Готовится на медицинский (Math + Bio)
- 2 часа в день на подготовку после школы
- Андроид-смартфон + старый ноутбук родителей
- Хочет чек-лист «что я уже знаю / не знаю», не любит длинные видео
- Решение: AlemPrep даёт прогресс по темам + быстрые сессии по 15-20 мин

**Даулет, 18, Алматы, второй год пересдача**
- В прошлом году получил 105 баллов, нужен 130
- Слабые места — стереометрия и текстовые задачи
- Готов платить за качественную подготовку до 5 000 ₸/мес
- Решение: AlemPrep даёт точечную работу со слабыми темами + mock exam'ы

**Айзере, 16, Астана, 10 класс**
- Начинает готовиться рано
- Геймификация работает (стрики, бейджи)
- TikTok / Instagram активна, готова шарить результаты
- Решение: AlemPrep даёт стрики, потом achievements (Шаг 9)

### 2.4 Pain points

1. **Дорого:** репетитор 5 000-15 000 ₸/час, годовая подготовка может стоить 500 000+ ₸
2. **Фрагментарно:** PDF от учителей + Telegram-каналы разного качества + старые сборники
3. **Не системно:** нет прогресса, нет понятной траектории
4. **Кaз. язык бедно:** 80% контента только на русском
5. **Mock exam'ы редко:** официальные пробники 1-2 раза в год
6. **UX устаревший:** существующие платформы — гигантские формы, реклама, медленно

## 3. Конкурентный ландшафт

### 3.1 Прямые конкуренты

| Платформа | Цена | Контент | UX | Слабые места |
|---|---|---|---|---|
| **Daryn.online** | 30 000 ₸/мес | Полный | Старый | Дорого, только рус, медленно |
| **Daryn.kz** (гос) | Бесплатно | Ограничен | Очень слабый | Никто не пользуется |
| **MathPracticeUNT** (TG бот) | Бесплатно | Узкий | Бот | Только математика, нет прогресса |
| **Bilim Land** | 3 990 ₸/мес | Школа+ЕНТ | Средний | Реклама, общеобразовательный |
| **Telegram-каналы** | Бесплатно | Разный | N/A | Нет системы, нет прогресса |
| **Khan Academy** | Бесплатно | Не ЕНТ | Отличный | Не наш формат, нет каз |
| **Репетиторы** | 5-15K ₸/час | Индивидуально | N/A | Дорого, не масштабируется |

### 3.2 Косвенные конкуренты

- Школьные учителя (бесплатно но непрофессионально по части ЕНТ)
- YouTube-каналы по подготовке к ЕНТ
- Печатные сборники (Мектеп, Атамура)
- Курсы при университетах

### 3.3 Наша позиция

**Позиционирование:** *«Anki + Khan Academy для ЕНТ. Бесплатно навсегда. Казахский язык как первоклассный гражданин.»*

**Уникальная ценность:**
1. **Бесплатный + качественный** — обычно одно из двух
2. **Современный UX** — deep work, no ads, клавиатурные шорткаты
3. **Казахский язык нативно** — не приклеен через автоперевод
4. **Open source** — портфолио + community contribution
5. **Адаптивное обучение** — фокус на слабых темах (Шаг 8)

### 3.4 Барьеры входа для конкурентов

- **Контент** — год работы чтобы накопить базу из 1500+ задач
- **Бренд** — узнаваемость в школах через teacher accounts
- **Платформа** — техническая зрелость + uptime
- **Стрики/привычки** — пользователи привязываются к streak

### 3.5 Чем мы НЕ конкурируем

- Не делаем общешкольную программу (только ЕНТ)
- Не делаем видеолекции (текст + интерактив быстрее)
- Не делаем тьюторов в core продукте (только marketplace позже)
- Не делаем подготовку к SAT/IELTS (Казахстан only)

## 5. Roadmap по спринтам

Каждый спринт — 1-2 недели работы для одного разработчика (тебя). Объём калибруется под реальную скорость.

**Цель:** AlemPrep работает по публичному URL и доступен с любого устройства.

**Задачи:**
- [ ] Связать GitHub репо с Vercel (Free tier)
- [ ] Добавить env-переменные в Vercel dashboard
- [ ] Купить домен `alemprep.kz` или `.com` (~$10/год)
- [ ] Настроить DNS + SSL (Vercel сделает автоматически)
- [ ] Обновить Google OAuth: добавить production redirect URI
- [ ] Обновить Supabase URL Configuration → Site URL и Redirect URLs
- [ ] Auth callback: исправить хардкод `/ru` → определять по cookie/header
- [ ] Mobile responsiveness:
  - Sidebar → off-canvas drawer на mobile (Sheet из shadcn)
  - Touch-friendly target sizes (минимум 44px)
  - Адаптивная сетка карточек
- [ ] Sentry Free tier — error tracking
- [ ] Plausible Analytics (или Vercel Analytics free)
- [ ] OG-метатеги для шеринга в WhatsApp/Telegram
- [ ] favicon.ico, apple-touch-icon
- [ ] robots.txt + sitemap.xml
- [ ] README.md обновить с публичным URL

**Definition of Done:** одноклассник может зайти с телефона по ссылке и зарегистрироваться.

### Sprint 3 — AI Content Pipeline + Math Coverage (2 недели)

**Цель:** Полное покрытие математики ≥200 задач.

**Задачи:**
- [ ] CLI-скрипт `scripts/generate-questions.ts`:
  - Принимает: topic slug, count, difficulty range
  - Использует Anthropic API (Claude Sonnet)
  - Промпт-шаблон: «Вот задача из ЕНТ [JSON]. Сгенерируй N аналогичных, меняя числа и контекст, сохраняя математическую сложность. Верни JSON с полями stem/options/correct/explanation»
  - Сохраняет в БД с `is_published = false`
- [ ] Admin-страница `/admin/review` (защищена через `is_admin = true` в profiles):
  - Список неопубликованных задач
  - Можно править stem, options, correct
  - Кнопка «Опубликовать» меняет `is_published = true`
- [ ] Для каждой из 11 оставшихся математических тем (алгебра, уравнения, функции, тригонометрия, прогрессии, планиметрия, стереометрия, производная, комбинаторика, статистика, текстовые задачи):
  - Взять 2-3 эталонные задачи из пробника НЦТ
  - Прогнать через генератор N=15
  - Ручная проверка
  - Публикация
- [ ] Тэги задач: тип задачи, подтема, типичные ошибки
- [ ] **Цель:** ≥220 задач в production, все ≥1 раз проверены человеком

**Definition of Done:** все 12 тем математики имеют ≥15 задач каждая.

### Sprint 4 — Progress Page + Analytics (1 неделя)

**Цель:** Ученик видит свою историю и точки роста.

**Задачи:**
- [ ] Установить `recharts` для графиков
- [ ] Materialized view `user_topic_stats(user_id, topic_id, attempts, correct, avg_time)` + cron на обновление каждую ночь
- [ ] Страница `/progress`:
  - **Activity heatmap** — последние 12 недель, как у GitHub
  - **Сильные/слабые темы** — горизонтальный bar chart, % правильных по темам
  - **История стрика** — текущий, максимальный, календарь активности
  - **Последние сессии** — список с датой, темой, результатом
  - **Best stats** — самая быстрая правильная задача, самый длинный focus
- [ ] Дашборд: добавить блок «Подтянуть слабые темы» — 3 темы с худшим %
- [ ] Web Push notifications (опционально):
  - Запрос разрешения после второй сессии
  - Reminder в 19:00 если стрик в опасности

**Definition of Done:** ученик зайдя на `/progress` видит личный график активности и понимает где он слаб.

## 6. Эволюция технической архитектуры

### 6.1 Сейчас (≤1 000 пользователей)

```
[User Browser]
     ↓
[Vercel Edge — Next.js SSR + Static]
     ↓
[Supabase Frankfurt (Free)]
  ├ Postgres 15 (500 MB)
  ├ Auth + RLS
  └ Storage (1 GB)
```

Cost: $0/мес.

### 6.2 5 000 пользователей

Тот же стек, всё ещё free tier. Добавляем:
- Cloudflare CDN в front (бесплатно) — кэширование статики + DDoS защита
- Sentry (5K events/mo free)
- Plausible Analytics ($9/mo) или Vercel Analytics ($10/mo)

Cost: $0-20/мес.

### 6.3 20 000 пользователей

- Supabase Pro ($25/mo) — 8 GB DB, 100K MAU, daily backups, point-in-time recovery
- Vercel Pro ($20/mo) — увеличенные лимиты, лучшие build minutes
- Upstash Redis ($10/mo) — кэш hot queries (текущие задачи, leaderboards)
- Уровень оптимизации:
  - Materialized views для статистики
  - DB indexes аудит
  - React Server Components везде где можно

Cost: ~$60/мес.

### 6.4 100 000 пользователей

- Supabase Team plan ($599/mo) или собственная Postgres (Hetzner Cloud €30/mo + RDS reads)
- Vercel Enterprise или собственный Node.js на Hetzner
- Redis cluster
- Read replicas Postgres
- Edge functions для критичных эндпоинтов
- CDN для изображений (Cloudflare R2 или Bunny)

Cost: $200-1000/мес — нужна выручка.

### 6.5 Архитектурные решения уже принятые

✅ **JSONB для question body** — отлично масштабируется на миллионы записей, гибко для новых типов.

✅ **Server Components везде по умолчанию** — меньше JS отправляется в браузер, лучше SEO.

✅ **Полиморфный question schema** — добавить новый тип = 1 case в renderer + миграция enum.

✅ **next-intl с URL prefix** — каждая локаль это отдельный URL, SEO дружелюбно, шарится правильно.

✅ **RLS policies в БД** — security at the source, нельзя случайно слить чужие данные.

### 6.6 Решения которые надо принять

| Вопрос | Опции | Рекомендация |
|---|---|---|
| Поиск задач (когда будет 1000+) | Postgres FTS / Algolia / Typesense | Postgres FTS до 10K, потом Typesense |
| Изображения в задачах | Storage Supabase / Cloudflare R2 | Supabase до Pro, потом R2 |
| Аналитика событий | Posthog self-hosted / Plausible | Plausible для traffic + Postgres events для product |
| AI-генератор: где запускать | Edge function / Local CLI / GitHub Actions | Local CLI, потом GH Action на cron |
| Платежи | Kaspi / Stripe / Paddle | Kaspi для КЗ обязательно, потом Stripe для международки |

## 7. Контент-стратегия

Самое сложное в edtech — контент. Технология решается за месяц, контент — годами.

### 7.1 Фазы

**Фаза 0 (текущая) — Manual seed.** 9 задач написаны вручную на основе пробника НЦТ. Цель — проверить инфраструктуру.

**Фаза 1 (Sprint 3-5) — AI-assisted.**
- Источник: 41 скриншот из реального пробника НЦТ + 200+ задач из открытых сборников
- Каждая задача → промпт Claude → 5-10 вариантов с разными числами/контекстом
- Ручная проверка каждой
- Цель: 500 задач к концу Q3 2026

**Фаза 2 (Sprint 6-7) — Расширение на физику и информатику.**
- Тот же подход + специфичные подсказки промпта для каждого предмета
- Цель: 1000 задач к концу Q4 2026

**Фаза 3 (Sprint 10+) — Community contributions.**
- Учителя через teacher accounts могут submit'ить свои задачи
- Reward: free premium для верифицированных авторов
- Moderation queue + reputation система

**Фаза 4 — Real exam coverage.**
- Сразу после каждого ЕНТ (август): получить актуальные задачи (через учителей, форумы)
- Переписать → опубликовать в течение 7 дней
- Marketing moment: «ЕНТ 2027 уже в AlemPrep»

### 7.2 Quality bar

Каждая задача публикуется только если:
- ✅ Условие сформулировано однозначно
- ✅ Ровно один (для single) или явное число (для multi) правильных ответов
- ✅ Дистракторы правдоподобны (типичные ошибки решения)
- ✅ Объяснение разбивает решение на ≥2 шага
- ✅ LaTeX рендерится без ошибок
- ✅ Сложность откалибрована (после первых 50 решений сравниваем с заявленной)
- ✅ Соответствует теме по спецификации НЦТ

### 7.3 Тэгирование

Каждая задача имеет:
- `topic_id` — основная тема
- `subtopic` — подтема (например для алгебры: «формулы сокращённого умножения»)
- `difficulty` — 1-5, начальная оценка от автора
- `calibrated_difficulty` — пересчитывается еженедельно по `% correct` (если >80% — снижаем, <20% — повышаем)
- `cognitive_skills` — что проверяет (вычисление / применение формулы / multi-step / интуиция)
- `common_errors` — массив типичных ошибок для analytics

### 7.4 Copyright

**Никогда не публикуем дословный текст задачи из официального пробника НЦТ.** Только переформулированные с измененными значениями. Источник всегда `source = 'ai_rewritten'`.

В Terms of Service явно указываем: «Все задачи являются оригинальными интеллектуальными работами AlemPrep либо переработанными общедоступными задачами с указанием изменений.»

## 14. Принципы дизайна и кода

Чтобы расти быстро без накопления долга, держим себя в рамках:

1. **TypeScript strict, всегда.** No `any`, no `@ts-ignore` без комментария.
2. **Server Components by default.** Client component только когда нужен state/effects.
3. **RLS обязательны.** Каждая новая таблица — сначала RLS, потом код.
4. **Server actions для записи.** Прямые fetch к Supabase из client только для чтения.
5. **Тесты для критических путей.** Auth, recordAttempt, calculate streak — без них не релизить.
6. **Перед merge: typecheck + lint должны пройти.** CI настроить в Sprint 2.
7. **Изменения схемы только через миграции.** Никаких ручных правок в Supabase Studio в проде.
8. **Каждая фича начинается с user story.** «Как [persona], я хочу [действие], чтобы [результат].»
9. **Sprints начинаются с дизайна, заканчиваются ретроспективой.**
10. **Документировать.** ROADMAP обновляется каждый sprint. README актуальный. Inline comments для нетривиальной логики.
